## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (0.875rem) | 400 | 1.5 |
| Label / Button | 16px (1rem) | 600 | 1.4 |
| Section Heading | 20px (1.25rem) | 600 | 1.2 |
| Page Heading | 24px (1.5rem) | 600 | 1.2 |

Source: Existing global.css patterns. Body at 0.875rem is the project standard for form text and descriptions. Buttons use 1rem/600. Page heading uses size (24px vs 20px) to differentiate from section headings -- a third weight is unnecessary.

| Accent (10%) | `--primary` | #1565c0 | See reserved list below |
| Destructive | `--danger` | #d32f2f | Error messages only |
| Success | `--success` | #2e7d32 | Success state of analysis completion |

| Warning background | `--warning-bg` | #fff3e0 | Partial analysis warning banner background |
| Warning border | `--warning-border` | #ffcc80 | Partial analysis warning banner border |

All tokens above already exist in global.css (including `--warning-bg` and `--warning-border` which are defined in both light and dark theme variants).

Accent (`--primary`) reserved for:
- "Generate Report" button in AutoLab results (gradient background)
- "Generate Report" button on report page (gradient background)
- "Run Analysis" button on report standalone mode
- Text links (retry, go back)
- Active nav item indicator
- Focus rings on form inputs

**Focal point:** The "Generate Report" button is the primary visual anchor on both the AutoLab results view and the report page. Its accent gradient background, full-width placement, and elevated shadow make it the dominant interactive element the user's eye should land on.

## Component Contracts

### 1. "Generate Report" Button (AutoLab Results Page)

**Placement:** Below the inline AI chat panel (bottom of AutoLab results), full width of the results container.

**Visual spec:**
- Background: `linear-gradient(135deg, var(--primary) 0%, #1976d2 100%)` (matches existing Generate Report button on report page)
- Text: white, 16px (1rem), weight 600
- Padding: 16px 32px
- Border radius: 8px (`--radius-sm`)
- Box shadow: `0 2px 8px rgba(21, 101, 192, 0.3)`, hover: `0 4px 12px rgba(21, 101, 192, 0.4)`
- Margin top: 24px (separation from chat panel)
- Icon prefix: document emoji or none (match project emoji convention)

**States:**
| State | Visual |
|-------|--------|
| Default | Blue gradient, pointer cursor |
| Hover | Elevated shadow, slight brightness increase |
| Disabled (analysis incomplete) | Not rendered -- button only appears after successful analysis |

**Interaction:** `onClick` stores results in AnalysisContext, calls `useNavigate('/report')`.

### 2. Report Page Standalone Analysis Section

**Placement:** Top of report page, before context form. Visible only when no analysis data is loaded (neither from AutoLab context nor from standalone run).

**Layout:**
- Card container: `--surface` background, `--border` 1px border, `--radius` border-radius, `--shadow-sm` shadow
- Padding: 24px
- Section heading: "Run Analysis" at 20px/600
- Subtext: Body copy explaining user can upload data and run analysis directly

**Sub-components:**
- File upload dropzone: Reuses visual pattern from existing ReportBeta file upload (dashed border, centered text)
- Instructions textarea: Standard textarea, 4 rows minimum, placeholder text
- "Run Analysis" button: Same gradient style as Generate Report button, full width

**States:**
| State | Visual |
|-------|--------|
| Empty (no file) | Upload prompt visible, Run Analysis disabled (gray background `#bbb`) |
| File uploaded | Filename shown, instructions textarea active, Run Analysis enabled |
| Running | Button text "Analyzing...", disabled, spinner or pulsing animation |
| Complete | Analysis section collapses or transforms to show "Analysis loaded" confirmation, context form becomes active |
| Error | Red error text below button: `var(--danger)` color, 14px, with retry link |

### 3. Pre-filled Context Form (AutoLab -> Report flow)

**Behavior:** When arriving from AutoLab, pre-populate fields that can be inferred:
- Experiment title: from AutoLab instructions or filename
- Data context: auto-filled from analysis parameters summary

**Visual indicator:** Pre-filled fields show a subtle info note: "Pre-filled from AutoLab analysis" in `var(--text-muted)` at 14px (0.875rem), below the input. The muted color provides sufficient visual de-emphasis without requiring a size reduction below the declared body size. Disappears on user edit.

### 4. Partial Analysis Warning Banner

**Placement:** Below the analysis summary / above the context form, when analysis data is incomplete.

**Visual spec:**
- Background: `var(--warning-bg)` (#fff3e0)
- Border: 1px solid `var(--warning-border)` (#ffcc80)
- Border radius: 8px
- Padding: 16px
- Icon: warning emoji
- Text: 14px (0.875rem), weight 400, color `var(--text)`
- Copy pattern: "[Section] not available -- will be skipped in the report."

**Examples:**
- "N-sigma comparison not available -- this section will be skipped in the report."
- "Fit analysis incomplete -- parameter table and fit plots will not appear in the report."

### 5. Nav Bar (BETA Badge Removal)

**Change:** Remove `badge: 'BETA'` from the Report nav item in `App.tsx`. No other nav changes.

## Interaction Contracts

### Navigation Flow: AutoLab -> Report

1. User completes AutoLab analysis (results render)
2. User scrolls to bottom of results, sees "Generate Report" button
3. User clicks button
4. App stores `autolabResults` and `plotImages` in AnalysisContext (already implemented)
5. App navigates to `/report` via `useNavigate`
6. Report page detects data in context, skips standalone analysis section
7. Context form appears with pre-filled fields
8. User fills remaining fields (partner name, ID, course), clicks "Generate Report"

### Standalone Report Flow

1. User navigates directly to `/report` (no AutoLab data in context)
2. Report page shows standalone analysis section at top
3. User uploads file + writes instructions
4. User clicks "Run Analysis" (calls `/api/autolab/run`)
5. On success: analysis section shows confirmation, context form activates
6. User fills context form, clicks "Generate Report"
7. Standard generation flow continues (follow-up questions, AI generation, preview/edit, PDF export)

### Edge Case: Partial Analysis

1. Analysis completes but some steps failed (e.g., n-sigma comparison errored)
2. Warning banner appears listing what is missing
3. User can still proceed to generate report
4. Missing data sections are omitted from generated report
5. No blocking -- user is informed but not prevented

### Edge Case: AI Generation Failure

1. Generation request fails
2. System retries once silently (no UI change during retry)
3. If retry fails: error message appears below button in `var(--danger)` color
4. "Try Again" text link appears below error message
5. User can click "Try Again" to reset and re-attempt
