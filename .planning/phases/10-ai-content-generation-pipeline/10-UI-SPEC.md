| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, gap between radio options, button row gaps, follow-up card button padding (vertical) |
| md | 16px | Default element spacing, form field gaps, input padding (vertical and horizontal), CTA button vertical padding |
| lg | 24px | Section padding inside cards, context form internal padding, button horizontal padding |
| xl | 32px | Gap between major sections (upload zone to context form to generate button), CTA button horizontal padding |
| 2xl | 48px | Page top/bottom padding |

Exceptions: none

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (0.875rem) | 400 | 1.6 |
| Label | 14px (0.875rem) | 400 | 1.4 |
| Heading | 20px (1.25rem) | 600 | 1.2 |
| Section title | 16px (1rem) | 600 | 1.4 |

All de-emphasized text (hints, subtexts, secondary labels) uses 14px/400 with `color: var(--text-muted)` for visual de-emphasis rather than a smaller font size.

Source: Matches existing ReportBeta.tsx inline styles (0.875rem body, 1.25rem headings, font-weight 600).

---

## Focal Point

The **Generate Report button** is the single visual anchor of this page. It is the only element using a gradient background (linear-gradient of var(--primary)), full-width layout, 600 weight text at 16px, and an elevated box-shadow. All other elements use flat backgrounds. The user's eye should travel: instruction upload -> context form -> Generate Report button.

## Interaction States

### Page Flow (top to bottom, single scrollable page)

1. **Page header** -- "Lab Report Export" + Beta badge (existing)
2. **Info banner** -- "Coming Soon" description (existing, keep as-is)
3. **Instruction file upload** -- Dropzone + extracted text textarea (existing from Phase 9)
4. **Context form** -- Title, subject, equipment, notes fields (NEW)
5. **Language toggle** -- Hebrew / English radio (NEW)
6. **Generate Report button** -- Primary CTA / focal point (NEW)
7. **Follow-up questions** -- Inline card with 1-3 questions (NEW, appears after clicking Generate)
8. **Loading state** -- Spinner during generation (NEW, replaces button area)
9. **Generation complete** -- Success card showing what was generated (NEW, replaces loading)

### State Transitions

| From | Action | To | Animation |
|------|--------|----|-----------|
| Idle (form visible) | Click "Generate Report" | Loading (analyze-context) | Button text changes to "Analyzing..." with spinner |
| Loading (analyze) | API returns questions | Follow-up questions card | Fade-in 200ms, scroll to card |
| Loading (analyze) | API returns 0 questions | Loading (generate) | Spinner + "Generating your report..." |
| Follow-up questions | Click "Generate Report" | Loading (generate) | Card fades out, loading replaces |
| Follow-up questions | Click "Generate Anyway" | Loading (generate) | Same as above |
| Loading (generate) | API returns sections | Generation complete | Fade-in success card |
| Any loading state | API error | Error state | Error message below form |

### Error Handling

| Error | Display |
|-------|---------|
| Network failure | Red text below form: "Failed to connect. Check your internet connection and try again." |
| API key missing (500) | Red text: "AI service is not configured. Contact the administrator." |
| Malformed AI response | Red text: "AI returned an unexpected response. Please try again." |
| Timeout (>30s) | Red text: "Generation timed out. Try providing more specific context and try again." |

Error styling: color var(--danger, #d32f2f), font-size 14px, margin-top 8px, with a "Try Again" link in var(--primary) color.

## Accessibility

| Concern | Contract |
|---------|----------|
| Form labels | Every input has an associated `<label>` element or `aria-label` |
| Error announcements | Error containers use `aria-live="polite"` (existing pattern from ReportBeta.tsx) |
| Focus management | After follow-up questions appear, focus moves to the first question input |
| Loading announcements | Loading container uses `aria-live="polite"` and `role="status"` |
| Language toggle | Implemented as `<fieldset>` with radio inputs, not custom div buttons |
| Keyboard navigation | All interactive elements reachable via Tab; Enter/Space activates buttons |
| Color contrast | All text meets WCAG AA (var(--text) #1a1a2e on var(--bg) #f8f9fc = 13.2:1) |
