    3. Create `backend/utils/pdf_renderer.py` with:

    a) `render_latex_for_pdf(latex_expr: str, display_mode: bool = False) -> str`:
    Uses `markdown_katex.extension.tex2html` with `{'no_inline_svg': True, 'insert_fonts_css': False}`.
    Falls back to a direct `subprocess.run` calling `npx katex` if markdown_katex import fails:
    ```python
    result = subprocess.run(
        ['npx', 'katex', '--no-throw-on-error'],
        input=latex_expr, capture_output=True, text=True, timeout=10
    )
    ```
    Returns the HTML string.

    b) `process_text_with_math(text: str) -> str`:
    Uses regex to find `$$...$$` (display math) and `$...$` (inline math) in text.
    For display math: wraps result in `<div class="math-display">`.
    For inline math: wraps result in `<span class="math-inline" dir="ltr">`.
    Display math regex: `r'\$\$([\s\S]*?)\$\$'` (process first).
    Inline math regex: `r'(?<!\$)\$(?!\$)([^$\n]+?)\$(?!\$)'` (process second).

    c) `generate_pdf(html_body: str) -> bytes`:
    - Reads `backend/templates/report_styles.css` content
    - Reads `backend/templates/report_base.html` as Jinja2 template (or simple string format)
    - Inserts CSS and body HTML into template
    - Calls `weasyprint.HTML(string=full_html, base_url=BACKEND_DIR).write_pdf(font_configuration=FontConfiguration())`
    - `BACKEND_DIR` is `os.path.dirname(os.path.dirname(os.path.abspath(__file__)))` (the `backend/` directory)
    - Returns PDF bytes

    d) `generate_test_pdf() -> bytes`:
