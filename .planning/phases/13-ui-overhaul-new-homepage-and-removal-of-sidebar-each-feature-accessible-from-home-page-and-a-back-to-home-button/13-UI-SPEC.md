## Color

All values reference existing dark theme tokens from `[data-theme="dark"]` in `global.css`.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--bg: #1a1a2e` | Page background, main content area |
| Secondary (30%) | `--surface: #16213e` | Tool cards, hero card background, elevated panels |
| Accent (10%) | `--primary: #4dd0e1` (teal) | See reserved list below |
| Secondary accent | `--accent: #42a5f5` (blue) | Tool card border-left colors, header gradient |
| Destructive | `--danger: #ef5350` | Not used in Phase 13 (no destructive actions) |

**Accent separation rule:** Teal (`--primary`) = interactive elements only. Blue (`--accent`) = structural decoration only — never on interactive elements.

### Accent reserved for (teal `#4dd0e1`):
- Hero AutoLab CTA arrow and hover glow
- Back-to-home button text color and hover underline
- Header logo hover glow effect
- Tool card hover border-left transition color

### Secondary accent reserved for (blue `#42a5f5`):
- Tool card default border-left color
- Header gradient background (existing tokens)
- Hero card subtle gradient accent

### Surface hierarchy (dark to light):
1. `--bg: #1a1a2e` — page background
2. `--surface: #16213e` — cards, panels at rest
3. `--surface-alt: #1e2a4a` — card hover state
4. `--surface-elevated: #243356` — hero card, elevated elements

### Text hierarchy:
1. `--text: #e8e8f0` — primary text, headings, card titles
2. `--text-secondary: #c0c0d4` — card descriptions, secondary info
3. `--text-muted: #a8a8c0` — footer text, disabled states

Source: Existing `[data-theme="dark"]` tokens in global.css lines 91-168.

## Interaction States

| Element | Default | Hover | Active/Focus |

## Dark Theme Activation Checklist

These are specific instructions for the executor, derived from RESEARCH.md findings:

1. **Activate:** Add `data-theme="dark"` to `<html>` in `frontend/index.html`
2. **Audit Plotly:** Check if `PlotWrapper.tsx` or chart components set `paper_bgcolor`/`plot_bgcolor`. If hardcoded white, change to `transparent` or matching dark token.
3. **Inline style hotspots:** Tool card `borderLeft` uses hardcoded hex colors from the `tools` array -- these are accent colors that should still look correct on dark backgrounds (they are intentionally bright). No change needed.
4. **Do NOT touch** `Sidebar.tsx` (AI chat panel) per D-11.
